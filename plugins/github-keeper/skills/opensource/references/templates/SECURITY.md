# Security Policy

## Supported versions

The latest released version on the default branch is supported. Fixes land in a
new release rather than as backports to older tags.

## Reporting a vulnerability

**Please do not open a public issue for a security problem.**

Use GitHub's private reporting:
[**Report a vulnerability**](https://github.com/{{OWNER}}/{{REPO}}/security/advisories/new).
This opens a confidential advisory visible only to the maintainers.

Please include a description and impact, the file/command involved with steps to
reproduce, and any suggested remediation. You can expect an acknowledgement within
a few days; once a fix ships, the advisory is published with credit unless you
prefer to remain anonymous.
